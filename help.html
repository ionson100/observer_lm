Наблюдатель за Локальным модулем Observer for Local Module

версия модуля: 2.****  |  API версии: 2

Позволяет инициализировать, наблюдать за логами, получать статус локального модуля. При первом старте программы настройки инициализируются по умолчанию.

⚠️ Важно: Вам нужно изменить Auth (по умолчанию строка Base64 от admin:admin) и Token, если вы собираетесь инициализировать модуль. Настройки находятся в директории ~/.config/obserberLm/settings.json (создаются при первом старте программы).

⚙️ Класс конфигурации (MySettings)

Ниже представлена структура класса настроек на C# с подробными комментариями:

    internal class MySettings
    {
        /// <summary>
        /// Путь к API локального модуля. Например: http://localhost:5995/api/v2/
        /// </summary>
        public string Url { get; set; }

        /// <summary>
        /// Строка в формате Base64 для авторизации Basic по умолчанию стоит admin:admin
        /// </summary>
        public string Auth { get; set; }
        
        /// <summary>
        /// Токен для авторизации по API
        /// </summary>
        public string Token { get; set; }
        
        /// <summary>
        /// Путь к папке логов
        /// </summary>
        public string FolderLog { get; set; }
        
        /// <summary>
        /// Количество строк в логах, 
        /// которые будут отображаться при открытии приложения, по умолчанию 100. 0 не допускается.
        /// </summary>
        public int Tail { get; set; } = 100;
    }

📄 Пример файла settings.json (по умолчанию)

Auth: admin:admin в Base64 → YWRtaW46YWRtaW4=
Tail: по умолчанию 100
FolderLog зависит от ОС (Windows / Linux). 

{
  "Url": "http://localhost:5995/api/v2/",
  "Auth": "YWRtaW46YWRtaW4=",
  "Token": "97486293-646b-463e-8199-48c37e36d605",
  "FolderLog": "C:\\Program Files\\Regime\\var\\log",
  "Tail": 100
}

Для Linux / macOS путь к папке логов может быть таким:

{
  "Url": "http://localhost:5995/api/v2/",
  "Auth": "YWRtaW46YWRtaW4=",
  "Token": "97486293-646b-463e-8199-48c37e36d605",
  "FolderLog": "/var/log/regime",
  "Tail": 100
}

🚀 Использование и особенности

  • Инициализация модуля: При первом запуске программа автоматически создает папку конфигурации ~/.config/obserberLm/ и файл settings.json со значениями по умолчанию. После этого вы можете отредактировать Auth и Token для корректной работы с API.
  • Наблюдение за логами: Утилита позволяет читать логи из директории, указанной в FolderLog. Параметр Tail определяет, сколько последних строк отображается при старте (по умолчанию 100).
  • Получение статуса: Через API версии 2 можно проверять состояние локального модуля, используя авторизацию Basic (Base64) или токен.
  • Безопасность: Рекомендуется сразу сменить стандартные учетные данные admin:admin и сгенерировать новый токен.

📌 Комментарии по настройке

Поле Описание Пример значения
Url Базовый URL API локального модуля (обязательно заканчивается на /api/v2/) http://localhost:5995/api/v2/
Auth Base64 строка для Basic Authentication. По умолчанию admin:admin → YWRtaW46YWRtaW4= dXNlcjpwYXNz
Token API токен для авторизации (если используется вместо Basic) 97486293-646b-463e-8199-48c37e36d605
FolderLog Абсолютный путь к каталогу с файлами логов C:\Program Files\Regime\var\log или /var/log/regime
Tail Количество строк лога для отображения при старте (не может быть 0) 100

📁 Структура конфигурационного файла

После первого запуска программы, настройки сохраняются в ~/.config/obserberLm/settings.json. Вы можете отредактировать этот файл вручную или через интерфейс. Обратите внимание, что путь FolderLog должен существовать и иметь права на чтение.

Пример корректного JSON (с комментариями в виде пояснений)

{
  // Адрес API локального модуля (версия 2)
  "Url": "http://localhost:5995/api/v2/",
  // Base64 строка для Basic Auth (admin:admin по умолчанию)
  "Auth": "YWRtaW46YWRtaW4=",
  // Токен, если вы используете токен-авторизацию (опционально)
  "Token": "97486293-646b-463e-8199-48c37e36d605",
  // Путь к директории с логами (зависит от ОС)
  "FolderLog": "C:\\Program Files\\Regime\\var\\log",
  // Количество строк для вывода (Tail)
  "Tail": 100
}

💡 Советы по использованию

  • При изменении Auth убедитесь, что строка корректно закодирована в Base64 (формат username:password).
  • Если вы используете Token, то Basic Auth может не требоваться, но логика определяется API модуля.
  • Параметр Tail не должен быть установлен в 0, иначе может возникнуть ошибка при попытке прочитать 0 строк.
  • Для корректного отображения логов на Linux убедитесь, что у процесса есть доступ к /var/log/regime.
  • При необходимости можно изменить URL на HTTPS, если локальный модуль поддерживает SSL.
